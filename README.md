# کِشورگُشایی 🏰

بازی آنلاین چندنفره (۲ تا ۵ نفر) تصرف سرزمین با سنگ‌کاغذقیچی.

## اجرا روی سیستم خودت (اختیاری، برای تست)
```bash
npm install
npm start
```
بعد آدرس `http://localhost:3000` رو باز کن.

## آپلود روی گیت‌هاب

1. یه ریپازیتوری جدید توی گیت‌هاب بساز (مثلاً `keshvar-goshaei`).
2. توی پوشه پروژه:
```bash
git init
git add .
git commit -m "اولین نسخه بازی کشورگشایی"
git branch -M main
git remote add origin https://github.com/USERNAME/keshvar-goshaei.git
git push -u origin main
```
(به‌جای `USERNAME` اسم کاربری خودت رو بذار.)

## دیپلوی روی Railway

1. برو به [railway.com](https://railway.com) و با گیت‌هاب لاگین کن.
2. **New Project** → **Deploy from GitHub repo** → ریپازیتوری `keshvar-goshaei` رو انتخاب کن.
3. Railway به‌طور خودکار `package.json` رو تشخیص می‌ده و با `npm install` و `npm start` بازی رو بالا میاره؛ نیازی به تنظیم دستی نیست.
4. بعد از اتمام دیپلوی، از تب **Settings → Networking** روی **Generate Domain** بزن تا یه آدرس عمومی مثل
   `https://keshvar-goshaei-production.up.railway.app`
   بگیری.
5. همون لینک رو برای دوستانت بفرست — همه از روی گوشی یا سیستم خودشون وصل می‌شن و توی لابی بازی کد ۶ رقمی اتاق رو باهم رد و بدل می‌کنن (یا از دکمه «کپی لینک دعوت» که کد رو خودش توی لینک می‌ذاره استفاده کن).

### نکته مهم درباره‌ی ماندگاری اطلاعات
وضعیت بازی‌ها فعلاً توی حافظه‌ی سرور نگه داشته می‌شه (نه دیتابیس)، یعنی:
- تا وقتی سرور روشنه همه‌چیز درست کار می‌کنه.
- اگه Railway سرویس رو ری‌استارت/ری‌دیپلوی کنه، بازی‌های درحال‌انجام پاک می‌شن.
- فقط باید **یک** اینستنس از سرویس بالا باشه (Replicas = 1)؛ چون حافظه بین چند اینستنس مشترک نیست.

اگه بعداً خواستی بازی‌ها بعد از ری‌استارت هم بمونن، می‌تونیم یه پلاگین دیتابیس رایگان Railway (مثل Redis یا Postgres) اضافه کنیم — کافیه بگی.
